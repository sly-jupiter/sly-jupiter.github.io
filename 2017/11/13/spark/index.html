Spark 入门学习资料整理

发表于 2017-11-13 | 分类于 大数据 ， Spark

什么是Spark

Spark 是一个用来实现快速而通用的集群计算的平台。
Spark，是一种“one stack to rule them all 的大数据计算框架，期望使用一个技术堆栈完美的解决大数据领域的各种计算任务。
Apache官方定义：通用的大数据快速处理引擎

Spark的架构

Spark SQL

Spark SQL是Spark 用来操作结构化数据的程序包。
通过Spark SQL，我们可以使用SQL或者 Apache Hive 版本的 SQL 方言（HQL）来查询数据。Spark SQL支持多种数据源，比如 Hive表、Parquet以及JSON 等。除了为 Spark 提供了一个 SQL 接口，Spark SQL还支持开发者将 SQL 和传统的 RDD 编程的数据操作方式相结合，不论是使用Python、Java还是 Scala，开发者都可以在单个的应用中同时使用 SQL 和复杂的数据分析。通过与 Spark所提供的丰富的计算环境进行如此紧密的结合，Spark SQL 得以从其他开源数据仓库工具中脱颖而出。Spark SQL是在 Spark 1.0 中被引入的。
在Spark SQL之前，加州大学伯克利分校曾经尝试修改Apache Hive以使其运行在 Spark上，当时的项目叫作Shark。现在，由于 Spark SQL与 Spark引擎和API 的结合更紧密，Shark已经被Spark SQL所取代。

Spark Streaming

Spark Streaming是Spark 提供的对实时数据进行流式计算的组件。比如生产环境中的网页服务器日志，或是网络服务中用户提交的状态更新组成的消息队列，都是数据流。SparkStreaming 提供了用来操作数据流的 API，并且与 Spark Core 中的 RDD API 高度对应。这样一来，程序员编写应用时的学习门槛就得以降低，不论是操作内存或硬盘中的数据，还是操作实时数据流，程序员都更能应对自如。从底层设计来看，Spark Streaming 支持与
Spark Core 同级别的容错性、吞吐量以及可伸缩性。

GraphX

GraphX是用来操作图（比如社交网络的朋友关系图）的程序库，可以进行并行的图计算。与 Spark Streaming 和 Spark SQL 类似，GraphX 也扩展了 Spark 的 RDD API，能用来创建一个顶点和边都包含任意属性的有向图。GraphX 还支持针对图的各种操作（比如进行图分割的 subgraph 和操作所有顶点的 mapVertices），以及一些常用图算法（比如 PageRank和三角计数）。

MLlib

Spark 中还包含一个提供常见的机器学习（ML）功能的程序库，叫作 MLlib。MLlib 提供了很多种机器学习算法，包括分类、回归、聚类、协同过滤等，还提供了模型评估、数据导入等额外的支持功能。MLlib 还提供了一些更底层的机器学习原语，包括一个通用的梯度下降优化算法。所有这些方法都被设计为可以在集群上轻松伸缩的架构。

Spark Core

Spark Core实现了 Spark 的基本功能，包含任务调度、内存管理、错误恢复、与存储系统交互等模块。Spark Core 中还包含了对弹性分布式数据集（resilient distributed dataset，简称 RDD）的 API 定义。RDD 表示分布在多个计算节点上可以并行操作的元素集合，是Spark 主要的编程抽象。Spark Core 提供了创建和操作这些集合的多个 API。

SparkContext

DriverApplication的执行与输出都是通过SparkContext来完成的，在正式提交Application之前，首先需要初始化SparkContext。 SparkContext内置的DAGScheduler负责创建Job，将DAG中的RDD划分到不同的Stage，提交Stage等功能。内置的TaskScheduler负责资源的申请、任务的提交及请求集群对任务的调度等工作。

存储体系

Spark优先考虑使用各节点的内存作为存储，当内存不足时才会考虑使用磁盘，这极大地减少了磁盘I/O，提升了任务执行的效率，使得Spark适用于实时计算、流式计算等场景。

计算引擎

计算引擎由SparkContext中的DAGScheduler、RDD以及具体节点上的Executor负责执行的Map和Reduce任务组成

部署模式

由于单节点不足以提供足够的存储及计算能力，所以作为大数据处理的Spark在SparkContext的TaskScheduler组件中提供了对Standalone部署模式的实现和Yarn、Mesos等分布式资源管理系统的支持。通过使用Standalone、Yarn、Mesos等部署模式为Task分配计算资源，提高任务的并发执行效率。除了可用于实际生产环境的Standalone、Yarn、Mesos等部署模式外，Spark还提供了Local模式和local-cluster模式便于开发和调试。

Spark的优点

速度快

Spark基于内存进行计算（部分基于磁盘，例如shuffle）

易上手

Spark基于RDD的计算模型，MapReduce更加易于理解，易于上手去实现各种复杂的功能。例如topN、二次排序等，变得更加便捷。

通用性强

Spark提供了Spark RDD、Spark SQL、Spark Streaming等技术组件，可以一站式完成大数据的离线批处理、交互式查询、流式计算、机器学习、图计算等任务。

集成Hadoop

Hadoop+Spark高度集成，两者可以完美配合使用。Hadoop的hdfs、hive、hbase等负责存储，Yarn负责资源调度，Spark负责复杂大数据计算。Hadoop+Spark是一种doublewin的策略，

活跃度很高

Spark是Apache的顶级项目，目前有很多大公司大规模的使用中。

Spark与其他主流大数据处理工具的对比

Spark VS MapReduce

MapReduce可以完成的各种离线批处理功能，Spark RDD的核心编程都可以是实现，并且更好更容易，且运行速度更快（数倍、数十倍、数百倍）
MapReduce计算模型太死板，必须是map-reduce模式。比如就算是过滤之类的操作也要map-reduce，这样就必须要shuffle，然而shuffle非常消耗性能（shuffle基于磁盘读写）。Spark的shuffle虽然也基于磁盘读写，但是spark有大量的transformation操作是基于内存的，这样性能就大大提高了。
但是，spark不够完善，比如基于HBASE、HIVE作为高级批处理程序的输入输出，spark就没有MapReduce完善。

Spark SQL VS Hive

  1. Spark SQL并不能完全替代Hive。Hive是基于hdfs的数据仓库，提供了基于SQL模型的、针对存储了大数据的数据仓库进行分布式交互查询的查询引擎。

  2. 严格说，Spark SQL可以替代的是hive的查询引擎。Spark本身不提供存储。

  3. Spark SQL相较于hive查询引擎的一个优点就是速度快

  4. Spark SQL相较于hive的另一个优点就是支持大量的不同的数据源（像hive、json、jdbc、parquet等）

  5. Spark SQL基于RDD工作，与其他的spark组件整合使用，配合使用是吸纳很多复杂功能（例如：Spark SQL支持直接对hdfs文件执行sql语句）

  6. Spark SQL目前为止还不够完善。

Spark Streaming VS Storm topology

  1. 计算模型不一样
    spark streaming基于RDD，所以需要将一小段时间（例如：1秒）内的数据收集起来作为一个RDD，再针对这个batch的数据进行处理。是一种准实时的流计算框架。
    Storm却可以来一条数据立即对其进行处理计算。是真正的实时流计算框架

  2. Storm支持在分布式流计算程序（topology）在运行过程中，动态的调度并行度，从而动态的提高并发处理能力。Spark streaming无法动态调整并行度。

  3. spark streaming基于batch进行处理，相比于storm的单条数据处理，吞吐量高出数倍甚至数十倍以上。

  4. spark streaming可以与spark其他组件紧密结合，这种一站式的大数据处理功能和优势，storm无法匹敌。

  5. 综上
    实时性要求很高、实时数据量不稳定（例如在白天有高峰期）：选择storm
    实时性要求一般（例如允许一秒的准实时处理）、 不要求动态调整并行度：选择 spark streaming
